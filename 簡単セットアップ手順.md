# 🚀 最も簡単なセットアップ手順（コマンド不要）

## 🎯 段階的アプローチ

### ステップ1: まず基本版をデプロイ（データベースなし）

1. **GitHub.com** にアクセス
2. **New repository** をクリック
3. リポジトリ名を入力（例：`calendar-app`）
4. **Create repository** をクリック
5. 以下のファイルをアップロード：
   - `index.html`
   - `styles.css`
   - `script.js`（データベースなし版）
   - `_headers`
   - `_redirects`

### ステップ2: Cloudflare Pagesに接続

1. **Cloudflare Dashboard** にログイン
2. **Pages** をクリック
3. **Connect to Git** をクリック
4. GitHubリポジトリを選択
5. **Begin setup** をクリック
6. **Save and Deploy** をクリック

### ステップ3: 基本動作を確認

✅ この時点で基本的なカレンダーアプリが動作します
- イベントの作成・編集・削除
- ローカルストレージでの保存
- レスポンシブデザイン

### ステップ4: データベース版にアップグレード（オプション）

基本版が動作したら、データベース版にアップグレード：

1. GitHubリポジトリに以下を追加：
   - `script-db.js`
   - `functions/api/events.js`

2. `index.html`を編集：
   ```html
   <script src="script-db.js"></script>
   ```

3. KVストレージを設定：
   - Workers & Pages → KV → Create namespace: `CALENDAR_KV`
   - Pages → Settings → Functions → KV bindings を追加
     - Variable name: `CALENDAR_KV`
     - KV namespace: 作成したnamespaceを選択

## 🔧 トラブルシューティング

### デプロイが失敗する場合
1. まず基本版（`script.js`使用）でデプロイ
2. 動作確認後にデータベース版にアップグレード

### ファイルが見つからないエラー
- `index.html`がルートディレクトリにあるか確認
- ファイル名の大文字小文字を確認

### 403エラーが出る場合
- `_headers`ファイルが正しく配置されているか確認

## 📱 動作確認

基本版で以下が動作することを確認：
- ✅ カレンダー表示
- ✅ イベント作成
- ✅ イベント編集・削除
- ✅ ローカル保存（ブラウザ内）
- ✅ モバイル対応
