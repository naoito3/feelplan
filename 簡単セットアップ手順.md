# 🚀 最も簡単なセットアップ手順（コマンド不要）

## ステップ1: GitHubにアップロード

1. **GitHub.com** にアクセス
2. **New repository** をクリック
3. リポジトリ名を入力（例：`calendar-app`）
4. **Create repository** をクリック
5. すべてのファイルをアップロード（ドラッグ&ドロップ可能）

## ステップ2: Cloudflare Pagesに接続

1. **Cloudflare Dashboard** にログイン
2. **Pages** をクリック
3. **Connect to Git** をクリック
4. GitHubリポジトリを選択
5. **Begin setup** をクリック
6. **Save and Deploy** をクリック

## ステップ3: KVストレージを作成

1. Cloudflare Dashboard で **Workers & Pages** をクリック
2. **KV** タブをクリック
3. **Create a namespace** をクリック
4. Namespace name: `CALENDAR_KV` と入力
5. **Add** をクリック

## ステップ4: KVをPagesに接続

1. **Pages** → あなたのプロジェクトをクリック
2. **Settings** → **Functions** をクリック
3. **KV namespace bindings** セクションで **Add binding** をクリック
4. Variable name: `CALENDAR_KV`
5. KV namespace: 先ほど作成した `CALENDAR_KV` を選択
6. **Save** をクリック

## ステップ5: 完了！

✅ これで完了です！あなたのカレンダーアプリが動作します。

**アクセス方法:**
- Cloudflare Pagesが提供するURL（例：`https://your-app.pages.dev`）

## 🔄 更新方法

ファイルを変更したい場合：
1. GitHubリポジトリでファイルを編集
2. **Commit changes** をクリック
3. Cloudflare Pagesが自動的に再デプロイ

## 📱 機能確認

- ✅ カレンダー表示
- ✅ イベント作成
- ✅ イベント編集・削除
- ✅ データの永続化
- ✅ モバイル対応

## 🆘 トラブルシューティング

### イベントが保存されない場合
1. Cloudflare Pages → Settings → Functions
2. KV namespace bindings が正しく設定されているか確認
3. Variable name が `CALENDAR_KV` になっているか確認

### サイトが表示されない場合
1. GitHubリポジトリに `index.html` があるか確認
2. Cloudflare Pages のデプロイが完了しているか確認
